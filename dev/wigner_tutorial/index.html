Wigner angles: SO(3) from Λ vs SU(2) from U

InstructionalDecayTrees tracks each instruction path with a 4×4 Lorentz matrix Λ (an SO(3,1) element in the (px, py, pz, E) basis) and a 2×2 SU(2) matrix U that carries spinor phase information.

For relative Wigner angles between two decay paths, call the single public API wigner_zyz. This notebook explains why a naive SO(3) decode of Λ alone is not enough when ψ sits on a 2π branch, and how the tracked SU(2) matrix fixes that.

using InstructionalDecayTrees
using FourVectors
using LinearAlgebra

const IDT = InstructionalDecayTrees
wrap2(a, b) = mod(a - b + π, 2π) - π
wrap4(a, b) = mod(a - b + 2π, 4π) - 2π
wrap4 (generic function with 1 method)

Public API: wigner_zyz

After executing two instruction paths, form the relative tracker Δ = tracker_other * inv(tracker_reference) and extract ZYZ Wigner angles:

p1 = FourVector(-0.1467, 0.2235, -0.7847; E = 2.0452)
p2 = FourVector(-0.0873, 0.1803, -0.5584; E = 0.7718)
p3 = FourVector(0.0056, -0.0349, 0.1413; E = 0.2017)
p4 = FourVector(0.2284, -0.3689, 1.2019; E = 2.2606)
objs = (p1, p2, p3, p4)

path_ref = (ToHelicityFrame((1, 2, 3)), ToHelicityFrameParticle2(3))
path_other = (
    ToHelicityFrameParticle2((2, 3, 4)),
    ToHelicityFrameParticle2((3, 4)),
    ToHelicityFrame((3,)),
)

cmp = compare_instruction_paths(path_ref, path_other, objs)
w = wigner_zyz(cmp.relative)
@assert abs(w.ϕ + 3.1181030111491173) < 5e-9
@assert abs(w.θ - 0.38580543795548133) < 5e-9
@assert abs(w.ψ - 9.42467967506533) < 5e-9

The relative transform here is a pure spatial rotation (ξ ≈ 0), which is the regime where SU(2) branch information matters for Wigner-angle comparisons.

Internal decoders (tutorial only)

The package also implements two internal decoders used below. They are not exported; access them as InstructionalDecayTrees._wigner_zyz_so3 and _wigner_zyz_su2 when studying the implementation.

SO(3): Λ loses the 2π spinor branch

Build a pure ZYZ rotation and flip the sign of U (same Λ, opposite SU(2) double cover):

ϕ0, θ0, ψ0 = 0.4, 1.1, 0.7
Λ = IDT._rz_xyze(ϕ0) * IDT._ry_xyze(θ0) * IDT._rz_xyze(ψ0)
U_plus = IDT._build_su2(0.0, 0.0, 0.0, ϕ0, θ0, ψ0)
U_minus = -U_plus

t_plus = LorentzTracker(Λ, U_plus)
t_minus = LorentzTracker(Λ, U_minus)

so3_plus = IDT._wigner_zyz_so3(t_plus)
so3_minus = IDT._wigner_zyz_so3(t_minus)
su2_plus = IDT._wigner_zyz_su2(t_plus)
su2_minus = IDT._wigner_zyz_su2(t_minus)
pub_plus = wigner_zyz(t_plus)
pub_minus = wigner_zyz(t_minus)

@assert so3_plus.ϕ ≈ so3_minus.ϕ
@assert so3_plus.θ ≈ so3_minus.θ
@assert so3_plus.ψ ≈ so3_minus.ψ   # SO(3) cannot see the sign flip

@assert abs(wrap4(su2_minus.ψ, su2_plus.ψ + 2π)) < 1e-12
@assert abs(wrap4(pub_minus.ψ, pub_plus.ψ + 2π)) < 1e-12
@assert pub_plus.ψ ≈ ψ0
@assert pub_minus.ψ ≈ ψ0 + 2π

SO(3) reports the same Euler triple for both trackers because Λ is identical. SU(2) and the public wigner_zyz shift ψ by 2π when U → -U, which is required for strict agreement with the decayangle cross-check on extended ψ intervals.

Agreement on consistent branches

For a single SU(2) branch, SO(3), SU(2), and wigner_zyz agree modulo the expected 2π wraps:

for (ϕ0, θ0, ψ0) in ((0.4, 1.1, 0.7), (-2.3, 0.9, 2.6), (1.9, 2.2, -1.4))
    Λ = IDT._rz_xyze(ϕ0) * IDT._ry_xyze(θ0) * IDT._rz_xyze(ψ0)
    U = IDT._build_su2(0.0, 0.0, 0.0, ϕ0, θ0, ψ0)
    t = LorentzTracker(Λ, U)
    a_so3 = IDT._wigner_zyz_so3(t)
    a_su2 = IDT._wigner_zyz_su2(t)
    a_pub = wigner_zyz(t)
    @assert abs(wrap2(a_so3.ϕ, a_su2.ϕ)) < 1e-12
    @assert abs(wrap2(a_so3.θ, a_su2.θ)) < 1e-12
    @assert abs(wrap4(a_so3.ψ, a_su2.ψ)) < 1e-12
    @assert abs(wrap4(a_pub.ψ, a_su2.ψ)) < 1e-12
end

Takeaway

  • Use wigner_zyz in application and cross-check code.
  • Treat _wigner_zyz_so3 / _wigner_zyz_su2 as implementation details for understanding why both Λ and U are tracked.
